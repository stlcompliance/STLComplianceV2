# STL Compliance Page Constitutions

Page constitutions are binding archetype rules between platform constitutions and shared components. They make “unified UI” measurable across products.

## Required archetypes

- `app-shell.md`
- `dashboard.md`
- `index-list.md`
- `recordcreate.md`
- `recorddetail.md`
- `drawer-peek.md`
- `empty-loading-error-degraded.md`
- `cross-product-reference.md`
- `workflow-wizard.md`
- `report-print-export.md`
- `settings-preferences.md`
- `admin-permission-surface.md`

A page is not complete because it renders. It is complete only when it proves tenant and permission behavior, durable data behavior, all relevant states, light/dark readability, responsive/accessibility behavior, shared-pattern use, and regression tests.
