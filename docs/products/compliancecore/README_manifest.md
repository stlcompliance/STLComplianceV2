## Purpose

Compliance Core owns the regulatory meaning layer for STL Compliance / ARR:

- Governing bodies
- Jurisdictions
- Citations
- Regulatory sources
- Rulepacks
- Requirements
- Applicability logic
- Evidence requirements
- Exceptions and exemptions
- Compliance evaluations
- Evidence mapping suggestions
- Theoretical Situation Evaluation
- Controlled compliance catalogs
- Rulepack/version governance

Compliance Core does not own operational execution, people, assets, inventory, orders, routes, training completions, document files, quality holds, or reporting read models.
