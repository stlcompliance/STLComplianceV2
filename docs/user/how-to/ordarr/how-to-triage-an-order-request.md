# How to triage an order request

## Audience
Order coordinators, customer service users, and operations managers.

## Purpose
Confirm an order request is complete enough to launch product handoffs.

## Before You Start
- OrdArr access.
- Request selected.
- Customer eligibility, requested service, location, contact, and required fulfillment details.

## Steps
1. Open OrdArr.
2. Open the order request.
3. Review request status, readiness, customer eligibility, participants, and order lines.
4. Resolve missing information or return the request for clarification.
5. Confirm which execution products must receive handoffs.
6. Move the request to ready or launch the available handoff actions.

## What Happens Next
OrdArr coordinates readiness and handoff state. Each execution product owns its execution record after handoff acceptance.

## Troubleshooting
- If customer eligibility blocks triage, resolve the issue in CustomArr.
- If an execution product is unavailable, verify entitlement and handoff configuration.
- If required evidence is missing, attach or request the file through RecordArr.
