# How to send or track a purchase order

## Audience
SupplyArr buyers, managers, purchasing coordinators, and vendor coordination users.

## Purpose
Send procurement context to a vendor and monitor whether the order is acknowledged, fulfilled, or blocked.

## Before You Start
- SupplyArr access.
- Vendor order view or manage access.
- Vendor, item, destination, due date, and line-item details.

## Steps
1. Open SupplyArr.
2. Open **Purchasing**.
3. Open **Vendor orders**.
4. Select the vendor order.
5. Review vendor, line items, destination, due date, and current status.
6. Use the available send, share, or portal action if present for the tenant.
7. Monitor acknowledgement, fulfillment, exception, and receiving-related status.
8. Use notes or audit history to record operational follow-up when available.
9. Coordinate receiving in LoadArr when the goods arrive.

## What Happens Next
SupplyArr tracks vendor order state and procurement context. LoadArr handles physical receiving. Bills and payments remain outside STL Compliance.

## Troubleshooting
- If no send action is visible, use the vendor order detail as the internal tracking record and follow the configured vendor communication process.
- If goods arrived but the order still appears open, check LoadArr receiving status and SupplyArr status synchronization.
- If payment status is needed, check the external finance process.
