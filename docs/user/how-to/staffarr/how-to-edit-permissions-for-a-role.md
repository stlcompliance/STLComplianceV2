# How to edit permissions for a role

## Audience
StaffArr admins responsible for role templates and permission assignments.

## Purpose
Update the permissions attached to a StaffArr role template or role.

## Before You Start
- StaffArr access.
- Permission to manage roles and permissions.
- A role template or role to edit.
- Current product permission catalog.

## Steps
1. Open StaffArr.
2. Open **Roles** or **Permissions**.
3. Select the role or role template.
4. Review the current permission list.
5. Add or remove only the permissions needed for the role.
6. Save the permission changes.
7. Review permission audit history.
8. Ask affected users to refresh after projection updates.

## What Happens Next
StaffArr updates the role permissions and audit history. Affected users may gain or lose actions after projection updates.

## Troubleshooting
- If a product permission is missing from the catalog, refresh or sync the catalog before editing.
- If access does not change immediately, wait for permission projection or ask an admin to process it.
- Do not add broad admin permissions just to work around one blocked action.
