# How to create a site

## Audience
StaffArr admins who maintain internal physical places.

## Purpose
Create a StaffArr site so products can reference an internal place consistently.

## Before You Start
- StaffArr access.
- Permission to manage org structure or locations.
- Site name and site type.

## Steps
1. Open StaffArr.
2. Open **Org structure** or **Locations**.
3. Create a new org unit with unit type **Site**.
4. Choose a site type such as Office, Warehouse, Plant, Shop, Yard, Terminal, Customer embedded, Mixed use, or Other.
5. Set the status to **Active** when the site is ready for use.
6. Save the site.
7. Create child locations as needed under the site.

## What Happens Next
The site can be used by StaffArr placement and referenced by products such as MaintainArr, LoadArr, RoutArr, and TrainArr.

## Troubleshooting
- If a site is not visible elsewhere, confirm it is active.
- If a product needs a warehouse area or dock, create the location in StaffArr instead of duplicating it in the product.
- If a site is only planned, do not use it for active operations.
