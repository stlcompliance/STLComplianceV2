# How to assign a role

## Audience
StaffArr admins who manage user authority.

## Purpose
Assign a role template to a person with the correct scope.

## Before You Start
- StaffArr access.
- Permission to manage roles or people.
- A person record.
- A role template.
- Scope type such as Tenant, Site, Department, Team, or Position.

## Steps
1. Open StaffArr.
2. Open **People** and select the person.
3. Open the role or access area for the person.
4. Choose the role template.
5. Choose the scope type.
6. If the scope is not Tenant, choose the matching site, department, team, or position.
7. Set an expiration date only if the role is temporary.
8. Save the role assignment.
9. Review the permission projection or history if available.

## What Happens Next
StaffArr records the role assignment. Product permissions may update after projection refresh.

## Troubleshooting
- If the role template is missing, create or refresh the permission catalog first.
- If the person still cannot act, check projection history and product entitlement.
- If scope is wrong, remove or replace the assignment rather than adding broad access.
