# How to review incidents tied to a person

## Audience
Managers, HR users, and StaffArr admins with incident access.

## Purpose
Review a person's incident history and follow-up records.

## Before You Start
- StaffArr access.
- Permission to view the person and incident history.
- The person name or identifier.

## Steps
1. Open StaffArr.
2. Open **People** and select the person.
3. Find the **Personnel incidents** area.
4. Select an incident from the list.
5. Review status, category, severity, notes, attachments, and any TrainArr routing state.
6. Add a note or attachment if you have manage access.
7. Close the incident only when the required review is complete and the close action is available.

## What Happens Next
The incident remains part of the StaffArr personnel history. Related retraining, evidence, or corrective action remains owned by the appropriate product.

## Troubleshooting
- If incidents are not visible, you may not have permission or the person has no incidents.
- If attachments fail, check file type and connection.
- If an incident should trigger retraining, use the TrainArr routing action where available.
