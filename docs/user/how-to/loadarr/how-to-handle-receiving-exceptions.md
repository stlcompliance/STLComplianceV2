# How to handle receiving exceptions

## Audience
Warehouse receivers, supervisors, and quality users.

## Purpose
Record and resolve mismatches, damage, quarantine needs, or compliance blockers during receiving.

## Before You Start
- LoadArr access.
- Receiving exception create or resolve permission as needed.
- Receiving session and item details.
- Reason and evidence for the exception.

## Steps
1. Open LoadArr.
2. Open **Work** > **Receiving** or **Work** > **Exceptions**.
3. Select the receiving session or receiving exception.
4. Review expected quantity, received quantity, item condition, and blocked reason.
5. If creating a new exception, enter the mismatch or damage details.
6. Choose the next action: review, quarantine, hold, resolve as valid stock, or scrap when available.
7. Save the exception update.
8. Escalate quality-bearing issues when the workflow requires formal review.

## What Happens Next
LoadArr preserves the exception and updates inventory availability only through approved stock movement or resolution actions.

## Troubleshooting
- If the exception cannot be resolved, check permission and current status.
- If quality review is required, involve the quality or compliance lead.
- If quantities are wrong, review receiving history and stock ledger.
