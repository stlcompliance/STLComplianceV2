# How to handle delays or missed stops

## Audience
Dispatchers and drivers.

## Purpose
Record and review transportation exceptions that affect trip execution.

## Before You Start
- RoutArr access.
- Trip or stop record.
- Delay or missed stop details.

## Steps
1. Open RoutArr.
2. Open **Exceptions**, **Dispatch board**, or the trip detail.
3. Select the related trip or stop.
4. Create or update the exception using the available action.
5. Enter delay reason, missed stop details, and notes.
6. Notify affected users through the available workflow if needed.
7. Update trip or stop status after the issue is resolved.

## What Happens Next
RoutArr records the transportation exception and keeps the trip history audit-ready.

## Troubleshooting
- If exception creation is missing, check dispatcher or trip execution access.
- If customer communication is needed, use the customer workflow available to your tenant.
- If missed stop affects proof, review **Proof review**.
