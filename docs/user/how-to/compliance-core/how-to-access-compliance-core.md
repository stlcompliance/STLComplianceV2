# How to access Compliance Core

## Audience
Compliance users, reviewers, and admins.

## Purpose
Open Compliance Core and confirm the available pages.

## Before You Start
- A signed-in account.
- Compliance Core product entitlement.
- Role access such as compliance_admin, compliance_reviewer, tenant_member, tenant_admin, or platform admin depending on action.

## Steps
1. Sign in to STL Compliance.
2. Open the product switcher.
3. Select **Compliance Core**.
4. Confirm the Compliance dashboard opens.
5. Use the sidebar to open Registry, Mappings, Findings, Evaluation, Theoretical situation, Evidence mapping, Fact sources, Imports, Reports, Operator, or Admin.

## What Happens Next
Compliance Core opens with pages allowed by your entitlement and role. Actions such as import commit or rule-pack publication may require stronger access.

## Troubleshooting
- If Compliance Core is missing, check entitlement.
- If an action is missing, check compliance role or permission.
- If rule data is missing, check Registry, Imports, and Fact sources.
