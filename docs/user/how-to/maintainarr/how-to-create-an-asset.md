# How to Create an Asset

## Before you start
- You need MaintainArr asset-create permission.
- Have the minimum identity, class/type, ownership, location, and service-status details.
- StaffArr owns internal locations; use owner-backed selectors and Quick Create where permitted.

## Steps
1. Open MaintainArr **Assets** and select **Create asset**.
2. Complete **Basics**: display name/number, class, type, and operating status.
3. Add identity details such as year/make/model, serial/VIN, manufacturer, and reference taxonomy matches where applicable.
4. Select owning legal/organizational context and StaffArr site/location.
5. Add meters, service dates, components, warranties, and acquisition details only when available; these may be backfilled after the minimum record exists.
6. Review initial compliance/readiness questions and any missing-fact follow-ups from Compliance Core.
7. Attach or request evidence through RecordArr when needed.
8. Review duplicate candidates and select **Create asset**.

## What happens next
MaintainArr creates the durable asset, calculates initial readiness, and may create onboarding inspections, PM setup tasks, document requests, or backfill reminders.

## Troubleshooting
- Missing reference option: use owner-backed search/Quick Create rather than free text.
- Save fails: entered work remains visible; correct the field/error and retry.
- Asset is blocked: review the readiness panel for defects, inspection, document, qualification, assignment, or compliance requirements.
