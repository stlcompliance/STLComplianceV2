# How to update asset readiness

## Audience
Maintenance managers and users reviewing asset readiness.

## Purpose
Understand and update asset readiness through the records that affect it.

## Before You Start
- MaintainArr access.
- The asset record.
- Information about defects, work orders, inspections, holds, recalls, or compliance blockers.

## Steps
1. Open MaintainArr.
2. Open **Asset registry**.
3. Select the asset.
4. Review readiness detail and history.
5. Open related defects, work orders, inspections, recalls, downtime, or quality holds.
6. Resolve the owning blocker in the product that owns it.
7. Refresh or re-open the asset readiness view.
8. Confirm the readiness status changed as expected.

## What Happens Next
Asset readiness reflects MaintainArr-owned asset state plus relevant cross-product signals. It should not be edited as a standalone shortcut when blockers remain.

## Troubleshooting
- If readiness stays blocked, review blocker source and required clearing action.
- If inventory or quality holds are involved, check LoadArr or AssurArr.
- If compliance gates are involved, check Compliance Core.
