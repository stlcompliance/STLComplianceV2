# How to invite or create a user

## Audience
Tenant access managers, user administrators, and support users who manage account setup.

## Purpose
Create or invite a suite login while keeping the person record owned by StaffArr.

## Before You Start
- Access to user management.
- Email address and display name.
- Tenant membership details.
- StaffArr person record if the user represents an internal person.

## Steps
1. Open the suite access settings.
2. Open **Users**.
3. Find the create or invite user area.
4. Choose invite when the person should set their password through the invitation flow, or create when an authorized user is setting the initial account directly.
5. Enter the email and display name.
6. Set account flags such as active state or email verification only when they are appropriate.
7. Submit the invite or creation action.
8. Add the user to the correct tenant membership.
9. Assign product permissions through StaffArr roles instead of treating the login account as product authority.

## What Happens Next
NexArr owns the login, tenant membership, sessions, and suite access. StaffArr remains the owner of the person and product authority context.

## Troubleshooting
- If the user can sign in but cannot open a product, verify tenant membership and product entitlement.
- If the user can open a product but cannot take actions, verify StaffArr role or permission assignments.
- If the invite email is not received, confirm the email address and resend according to the tenant process.
