# How to troubleshoot login or entitlement problems

## Audience
Support users, tenant access managers, managers, and users checking missing access.

## Purpose
Separate sign-in problems, tenant membership, product entitlement, and product-specific permissions.

## Before You Start
- User email or account reference.
- Tenant name.
- Product the user expects to open.
- Any error message or screenshot.

## Steps
1. Confirm the user can reach the sign-in page and enter the correct email.
2. Confirm the account is active and belongs to the expected tenant.
3. Confirm the tenant has entitlement for the product.
4. Ask the user to switch to the correct tenant if they belong to more than one.
5. Check whether the product appears in the product switcher.
6. If the product appears but actions are missing, review StaffArr roles and product permissions.
7. Ask the user to refresh or sign out and back in after access changes.
8. Capture the error message and escalate if the account, tenant, entitlement, and permissions all look correct.

## What Happens Next
Most access issues resolve by correcting one layer: account, tenant membership, product entitlement, or product permissions.

## Troubleshooting
- If the user cannot sign in at all, start with account status, password, invitation, and tenant membership.
- If the product is missing, check entitlement and product switcher visibility.
- If only one action is missing, check product permissions and the record's status.
