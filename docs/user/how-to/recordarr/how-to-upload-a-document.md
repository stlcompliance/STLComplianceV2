# How to upload a document

## Audience
RecordArr users who add files or evidence.

## Purpose
Upload a document into RecordArr for storage, review, or evidence mapping.

## Before You Start
- RecordArr access.
- Upload or record creation access.
- The file and its document type or purpose.
- Related source record if applicable.

![RecordArr upload session showing a file drop area and required document metadata fields.](/screenshots/recordarr-upload.png "Complete document metadata during upload so the retained record can be found and reviewed later.")

## Steps
1. Open RecordArr.
2. Open **Uploads**, **Upload Sessions**, or **Capture**.
3. Start a new upload session or capture request where available.
4. Choose the file.
5. Enter required title, type, source product, or metadata.
6. Complete the upload session.
7. Open the created record to review metadata and status.

## What Happens Next
RecordArr owns the stored file, metadata, versions, retention, and access history.

## Troubleshooting
- If upload fails, check file size, connection, and required fields.
- If the document should be evidence, map it to the requirement after upload.
- If a product owns the original operational record, attach the document through that product when required.
