# MaintainArr User Guide

## What This Product Is For
MaintainArr is for asset master records, asset condition, readiness, inspections, defects, work orders, repairs, maintenance labor, downtime, maintenance evidence, recall tracking, and parts demand requests.

## Who Uses It
- maintenance managers
- technicians
- operators who report defects
- compliance users reviewing maintenance evidence

## Main Pages
- Maintenance readiness
- Asset registry
- PM programs
- Recalls
- Meters and readings
- Work orders
- Defects
- Inspection runs
- Inspection templates
- Maintenance parts
- Parts kits
- Maintenance history
- Downtime
- Workspace settings

## Main Records
- asset
- PM program
- meter reading
- work order
- defect
- inspection run
- inspection template
- parts demand line
- downtime event

## Common Workflows
- review asset readiness
- report defects
- create work orders
- execute inspections
- build inspection templates
- create PM programs
- request parts for a work order
- close work orders

## Permissions Usually Needed
- maintainarr_admin
- maintainarr_manager
- maintainarr_technician
- maintainarr.assets.create
- maintainarr.pm.manage
- maintainarr.inspections.manage
- maintainarr.pmPrograms.create

## Related Products
- StaffArr owns people and internal locations.
- TrainArr owns technician qualifications.
- LoadArr owns inventory availability and stock movement.
- SupplyArr owns item and vendor context.
- RecordArr stores retained maintenance evidence.

## Common Troubleshooting
- [Product not visible](../troubleshooting/product-not-visible.md)
- [Missing permission](../troubleshooting/missing-permission.md)
- If a page is visible but an action is disabled, check the record status and your role or permission assignment.
- Remember: MaintainArr does not own inventory stock truth, vendor master data, purchase approvals, staff location identity, dispatch execution, customer relationships, or financial asset ledger.
