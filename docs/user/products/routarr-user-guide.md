# RoutArr User Guide

## What This Product Is For
RoutArr is for routes, trips, dispatch plans, driver assignments, vehicle assignments, stop sequence, trip status, ETA updates, transportation exceptions, proof of pickup, proof of delivery, and dispatch audit trail.

## Who Uses It
- dispatchers
- drivers
- route managers
- customer-portal users where enabled

## Main Pages
- Dashboard
- Dispatch board
- Dispatch plans
- Route planner
- Driver portal
- Customer portal
- Trips
- Routes
- Stops
- Exceptions
- Proof review
- Dock appointments
- Load visibility
- Validation blockers
- Availability
- Calendar
- Settings

## Main Records
- trip
- route
- stop
- dispatch plan
- driver assignment
- vehicle assignment
- proof record
- dock appointment
- validation blocker

## Common Workflows
- create trips
- create routes
- assign drivers
- assign vehicles
- update trip execution
- review proof
- handle delays and exceptions

## Permissions Usually Needed
- routarr_admin
- routarr_manager
- routarr_dispatcher
- routarr_driver
- routarr.routes.create
- routarr.dispatch.assign
- routarr.trips.perform
- routarr.dispatch.manage

## Related Products
- StaffArr owns driver person records.
- TrainArr owns driver qualifications.
- MaintainArr owns vehicle readiness.
- LoadArr owns load and inventory readiness.
- RecordArr stores retained transport evidence.

## Common Troubleshooting
- [Product or feature not visible](../troubleshooting/product-or-feature-not-visible.md)
- [Missing permission](../troubleshooting/missing-permission.md)
- If a page is visible but an action is disabled, check the record status and your role or permission assignment.
- Remember: RoutArr does not own driver person master records, driver certifications, vehicle maintenance truth, inventory truth, warehouse receiving truth, customer master records, financial freight billing, ELD hardware, or telematics hardware.
