# StaffArr Admin Guide

## What This Role Does
A StaffArr administrator manages the internal people and place foundation: people, org structure, locations, roles, permissions, readiness, restrictions, and personnel incident history.

## What This Role Can Usually Access
- StaffArr People, Org structure, Locations, Roles, Permissions, Readiness, Restrictions, Incidents, Settings, and Admin when assigned.
- Incident management for roles such as tenant_admin, staffarr_admin, or hr_admin.
- Permission projection and role template tools where enabled.

## What This Role Usually Cannot Access
- Does not create training programs or issue certifications; TrainArr owns those.
- Does not own maintenance, inventory, dispatch, procurement, or stored document files.
- Does not grant platform-admin status or override product-owned action enforcement.

## Common Daily Tasks
- Create and update person records.
- Create org units, sites, departments, teams, positions, and locations.
- Assign roles and review permissions.
- Review readiness restrictions and certification snapshots.
- Report and review personnel incidents.

## Records This Role Works With
- person
- org unit
- site
- location
- role template
- permission projection
- readiness restriction
- personnel incident

## Notifications This Role May Receive
- Incident review tasks
- permission projection or worker warnings
- readiness blockers
- training acknowledgement receipts

## Common Issues
- Person not found because filters or tenant are wrong.
- Role assigned but permission projection has not refreshed yet.
- Location missing because StaffArr location setup is incomplete.
