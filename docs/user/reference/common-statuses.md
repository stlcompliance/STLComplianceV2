# Common Statuses

These statuses are confirmed in code, constitutions, or product UI. Products may show a smaller product-specific set.

## Platform lifecycle statuses
- draft
- submitted
- pending_review
- pending_approval
- approved
- rejected
- active
- scheduled
- in_progress
- blocked
- watch
- completed
- closed
- cancelled
- inactive
- archived
- superseded
- deleted

## StaffArr person and org statuses
- Applicant
- Pending start
- Onboarding
- Active
- Leave
- Suspended
- Terminated
- Inactive
- Archived
- Planned

## StaffArr incident values
- Reason categories: Safety, Conduct, Injury, Equipment, Training compliance, Policy, Other
- Severity: Low, Medium, High, Critical
- Incident status: open or closed where status update is available

## TrainArr assignment and qualification values
- Evaluation results: pass, fail, incomplete
- Signoff roles: trainee, trainer
- Qualification statuses shown in UI include issued and suspended; expiration and revocation actions are also present.

## SupplyArr purchase request statuses
- draft
- submitted
- approved
- rejected

## LoadArr examples
- Receiving condition: New, Pending inspection, Damaged, Quarantined
- Transfer types: Bin to bin, Zone to zone
- Exception states used in code include needs_review, needs_approval, needs_quarantine, resolved_valid_stock, resolved_scrap, blocked_by_compliance
