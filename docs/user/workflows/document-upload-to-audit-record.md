# Document Upload to Audit Record

## Purpose
Store a document and connect it to evidence or audit review.

## Who Participates
- Product user
- Records user
- Compliance user
- Auditor

## Starting Point
A document, photo, report, or evidence file needs to be retained.

## Main Steps
1. Upload or capture the document through the owning product or RecordArr.
2. Review metadata and source record.
3. Map the document to evidence or requirements where needed.
4. Include it in a package or report if audit review requires it.
5. Apply retention or legal hold controls where required.

## Products Involved
- RecordArr owns stored files and retention.
- Compliance Core owns evidence meaning.
- Owning products own operational records.
- ReportArr owns report generation before retention.

## Records Created or Updated
- file
- record
- metadata
- evidence mapping
- record package
- access log

## Where Users May Get Stuck
- Document uploaded but not mapped.
- Access denied.
- Wrong retention category.
- Evidence still missing because effective dates or requirement mapping are wrong.
