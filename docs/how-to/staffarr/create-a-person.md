# How to create a person

## Audience
HR, operations managers, and StaffArr admins

## Product
StaffArr

## Support Status
Supported by current UI/API

## Purpose
Create the canonical internal person record used across STL Compliance.

## Before You Start
- StaffArr owns internal people. Do not create local human identity records in other products.
- You need StaffArr permission to create people.

## Steps
1. Open StaffArr.
2. Open People.
3. Choose Create.
4. Enter the person identity and employment or worker details requested by the form.
5. Connect the person to the correct organization, manager, site, and location context when those fields are available.
6. Save the person.
7. Review the person detail page for status, placement, permissions, incidents, certifications, and history.

## What Happens Next
Other products reference the StaffArr person ID for assignments, training, dispatch, maintenance labor, incidents, and reporting.

## Troubleshooting
- If the person needs platform login, create or invite the NexArr user after the StaffArr person exists.
- If a department, site, or location is missing, create it in StaffArr first.
