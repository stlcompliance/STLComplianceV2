# How to create a location

## Audience
StaffArr admins and operations managers

## Product
StaffArr

## Support Status
Supported by current UI/API

## Purpose
Add an internal operational location such as a building, room, dock, yard, or addressable area.

## Before You Start
- StaffArr owns internal operational locations.
- LoadArr owns inventory truth and movement, but it should reference StaffArr locations.

## Steps
1. Open StaffArr.
2. Open Locations or Organization Structure.
3. Choose the appropriate location type or hierarchy area.
4. Create the location using the page create panel.
5. Enter the label, parent site or area, status, and operational details requested by the page.
6. Save the location.
7. Use the location in downstream workflows such as receiving, maintenance, dispatch, training, and reporting.

## What Happens Next
The location becomes the shared internal place reference for the suite.

## Troubleshooting
- If a product shows an outdated location label, verify it is consuming the current StaffArr location reference.
- If the location represents customer or vendor geography, confirm the owning product before adding it as an internal StaffArr location.
