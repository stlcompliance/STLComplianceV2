# How to edit role permissions

## Audience
StaffArr permission admins and product owners

## Product
StaffArr

## Support Status
Supported by current UI/API

## Purpose
Change what a StaffArr role can do inside one or more products.

## Before You Start
- You need role management access.
- Changing a role can affect everyone assigned to it.

## Steps
1. Open StaffArr.
2. Open Roles.
3. Select the role.
4. Review whether it is a standard role or custom role.
5. Open edit mode when editing is allowed.
6. Use the Permissions tab to set product/module access.
7. Use the Scope tab to confirm where the role applies.
8. Review Assignments to understand who is affected.
9. Save the role.
10. Review Audit after saving for the change record.

## What Happens Next
Products should apply the updated role permissions when they evaluate product-domain actions.

## Troubleshooting
- If a standard role cannot be edited, clone it or create a custom role if the UI allows that pattern.
- If a permission label is unclear, confirm the product module and workflow before granting broad access.
