# How to update controlled vocabulary

## Audience
Compliance admins and data stewards

## Product
Compliance Core

## Support Status
Supported by current API/docs with admin surface to confirm

## Purpose
Maintain consistent compliance terms used by rulepacks, mappings, and evidence requirements.

## Before You Start
- Compliance Core owns controlled compliance vocabulary.
- Vocabulary changes can affect search, mappings, imports, and evaluation.
- Exact UI labels for vocabulary editing should be confirmed in Admin or Settings.

## Steps
1. Open Compliance Core.
2. Open Admin or Settings.
3. Find the controlled vocabulary area.
4. Search for the term before creating a new one.
5. Create or edit the term, label, description, status, and relationships requested by the page.
6. Save the term.
7. Review affected mappings, imports, and evaluations.
8. Communicate vocabulary changes to product owners if terms appear in workflows or reports.

## What Happens Next
Compliance Core vocabulary becomes the shared language for compliance mapping and evaluation.

## Troubleshooting
- If the vocabulary editor is not visible, document the missing admin screen and use the approved backend/admin process only if one exists.
- Do not create duplicate terms to work around missing labels.
