# How to map uploaded documents to requirements

## Audience
Compliance admins and records coordinators

## Product
Compliance Core

## Support Status
Supported by current UI/API

## Purpose
Connect RecordArr documents or product evidence to Compliance Core evidence requirements.

## Before You Start
- Compliance Core owns evidence meaning and requirement mapping.
- RecordArr owns the stored file and retention/access history.
- Operational products own the event or record that produced the evidence.

## Steps
1. Open Compliance Core.
2. Open Evidence mapping.
3. Find the requirement, evidence type, product source, or uploaded document reference.
4. Review the requirement and source context.
5. Select or confirm the RecordArr record or product evidence reference.
6. Map the document to the requirement using the mapping controls.
7. Save the mapping.
8. Review reports, findings, or audit package impact.
9. Correct the stored file in RecordArr if the file itself is wrong.

## What Happens Next
Compliance Core records that the evidence satisfies or relates to a requirement. RecordArr remains the file owner.

## Troubleshooting
- If the document is missing, upload or capture it through RecordArr or the owning product first.
- If the rule requirement is wrong, update Compliance Core rule or evidence definitions before mapping.
