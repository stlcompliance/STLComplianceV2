# How to generate a report

## Audience
Report builders, managers, auditors, and admins

## Product
ReportArr

## Support Status
Supported by current UI/API

## Purpose
Run or build a report from governed report definitions and product read models.

## Before You Start
- ReportArr owns report definitions, dashboards, exports, and report history.
- ReportArr does not correct source data; corrections happen in the owning product.
- You need report builder or report access permission for builder actions.

## Steps
1. Open ReportArr.
2. Open Reports or Report Builder.
3. Choose an existing report or create a report definition if you have builder access.
4. Select the dataset and parameters requested by the page.
5. Add filters, sections, layout, and access policy when building a new report.
6. Preview the report.
7. Save or activate the definition when it is ready.
8. Run the report and review freshness, source, and results.

## What Happens Next
ReportArr generates a report view or artifact from approved datasets and snapshots. Source products remain the owners of corrections.

## Troubleshooting
- If the report is missing data, identify the source product and correct the source record there.
- If builder actions are disabled, verify ReportArr builder or admin permissions.
