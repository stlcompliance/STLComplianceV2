# How to review audit-ready records

## Audience
Auditors, compliance managers, report users, and records coordinators

## Product
ReportArr

## Support Status
Supported by current UI/API

## Purpose
Use ReportArr audit views to review audit readiness across products and records.

## Before You Start
- ReportArr owns audit dashboards and report views.
- RecordArr owns stored records and packages.
- Compliance Core owns evidence requirements and rule meaning.

## Steps
1. Open ReportArr.
2. Open Audit or the relevant dashboard/report.
3. Set tenant, date range, product, requirement, or status filters.
4. Review readiness, missing evidence, freshness, and source-product context.
5. Open linked records or packages in RecordArr when you need stored evidence.
6. Open Compliance Core when the issue is rule or evidence requirement interpretation.
7. Open the owning product when the issue is source data or operational completion.
8. Export the report only when needed and permitted.

## What Happens Next
ReportArr summarizes audit readiness. Corrections still happen in the owning product, RecordArr, or Compliance Core depending on the issue.

## Troubleshooting
- If a record is missing, check RecordArr and the source product.
- If a requirement is unexpected, review Compliance Core mappings.
- If the report is stale, check ReportArr data freshness before sharing.
