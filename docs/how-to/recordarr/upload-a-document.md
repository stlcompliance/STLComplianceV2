# How to upload a document

## Audience
Records coordinators and authorized product users

## Product
RecordArr

## Support Status
Supported by current UI/API

## Purpose
Create an upload session or record for a document that needs storage, retention, OCR, scan processing, or evidence mapping.

## Before You Start
- RecordArr owns stored files and document metadata.
- The source product owns the operational record that the document supports.
- Do not upload secrets or unrelated raw technical payloads.

## Steps
1. Open RecordArr.
2. Open Capture.
3. Create an Upload session.
4. Choose the source product, object type, object reference, and purpose.
5. Set scan, OCR, or manual review options only when the document needs them.
6. Upload or add the document through the session workflow.
7. Review upload status, scan processing, OCR review, and evidence mapping as needed.
8. Open Records to verify the stored record after processing.

## What Happens Next
RecordArr stores the document and tracks processing, review, evidence mapping, retention, and access history.

## Troubleshooting
- If upload processing stalls, check Upload Sessions, Uploads, Scan Processing, or OCR Review.
- If the source product reference is unknown, confirm it in the owning product before uploading.
