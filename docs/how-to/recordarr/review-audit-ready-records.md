# How to review audit-ready records

## Audience
Records coordinators, compliance users, and auditors

## Product
RecordArr

## Support Status
Supported by current UI/API

## Purpose
Assemble or review records that support an audit, closeout, training completion, receiving package, or compliance package.

## Before You Start
- RecordArr owns record packages, retention, holds, access history, and stored documents.
- Compliance Core owns evidence meaning and requirements.
- ReportArr owns reports and dashboards.

## Steps
1. Open RecordArr.
2. Open Packages or Record Packages.
3. Find the package by source product, purpose, audit, or record reference.
4. Open the package detail.
5. Review included records, evidence mappings, retention status, legal holds, redactions, access logs, and download availability.
6. Add or request missing records through RecordArr Capture or the owning product workflow.
7. Download the package only when permitted and needed.
8. Use Compliance Core or ReportArr to understand audit posture beyond the stored records.

## What Happens Next
RecordArr provides audit-ready stored evidence packages. It does not decide rule applicability or correct source product data.

## Troubleshooting
- If a required document is missing, capture it through RecordArr or the owning product.
- If evidence meaning is unclear, review Compliance Core evidence requirements.
