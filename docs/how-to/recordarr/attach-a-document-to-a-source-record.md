# How to attach a document to a source record

## Audience
Records coordinators and product users

## Product
RecordArr

## Support Status
Supported by current UI/API with source-product coordination

## Purpose
Connect a document to a person, asset, work order, incident, vendor, order, trip, or other product-owned record without moving ownership.

## Before You Start
- RecordArr owns the file and record metadata.
- The source product owns the operational record.
- Use a stable source product, object type, and object reference when attaching.

## Steps
1. Open RecordArr.
2. Open Capture.
3. Create or open an Upload session.
4. Set Source product to the product that owns the operational record.
5. Enter the object type and object reference from the source product.
6. Set purpose and processing options.
7. Upload the document.
8. Complete scan, OCR, or manual review if required.
9. Open the source product record to confirm the attachment or evidence reference appears when that product supports it.
10. Use RecordArr Packages or Evidence Mappings for audit assembly.

## What Happens Next
RecordArr stores the file and source reference. The source product continues to own the person, asset, work order, incident, vendor, order, or trip.

## Troubleshooting
- If the source product does not show the attachment, check whether that product has implemented RecordArr attachment display.
- If the object reference was entered incorrectly, correct the RecordArr source reference according to the available workflow instead of uploading a duplicate.
