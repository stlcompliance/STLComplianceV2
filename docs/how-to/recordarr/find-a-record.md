# How to find a record

## Audience
Records coordinators, auditors, managers, and product users

## Product
RecordArr

## Support Status
Supported by current UI/API

## Purpose
Search for stored records, evidence, documents, and record packages.

## Before You Start
- RecordArr owns stored files, record metadata, retention, access history, and controlled document lifecycle.
- You need permission to view the record.

## Steps
1. Open RecordArr.
2. Open Records.
3. Use Search records to search by record number, title, description, source, file, or tags.
4. Apply filters when available for source product, status, type, or date.
5. Open the record detail.
6. Review metadata, source references, evidence mapping, packages, retention, holds, and access history.
7. Download the file only when you have permission and a valid business reason.

## What Happens Next
RecordArr shows the stored record and controls access, retention, packages, and evidence references.

## Troubleshooting
- If a record is missing, check whether the owning product captured or uploaded it.
- If access is denied, ask a RecordArr administrator to review your permissions or access policy rather than duplicating the document elsewhere.
