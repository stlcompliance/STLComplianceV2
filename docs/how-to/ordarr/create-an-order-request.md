# How to create an order request

## Audience
Customer service users, order coordinators, and operations coordinators

## Product
OrdArr

## Support Status
Supported by product contract/docs

## Purpose
Create the OrdArr parent request that coordinates customer work across execution products.

## Before You Start
- OrdArr owns order and request orchestration.
- CustomArr owns customer truth.
- Execution products own execution truth.

## Steps
1. Open OrdArr.
2. Open Orders, Requests, or Intake.
3. Choose Create.
4. Add customer reference, request details, requested timing, participants, locations, and order lines.
5. Review readiness checks and missing information.
6. Save the request.
7. Move the request to triage when enough information is available.

## What Happens Next
OrdArr owns the request lifecycle. Handoffs can later create or reference execution records in the owning products.

## Troubleshooting
- If the customer is missing, resolve it in CustomArr.
- If a product target is unavailable, check destination operational status and OrdArr handoff configuration.
