# How to triage an order request

## Audience
Order coordinators, customer service users, and operations managers

## Product
OrdArr

## Support Status
Supported by product contract/docs

## Purpose
Confirm an order request is complete enough to launch execution handoffs.

## Before You Start
- Customer eligibility should be known from CustomArr.
- Required service, location, contact, and fulfillment details should be present.

## Steps
1. Open OrdArr.
2. Open the order request.
3. Review status, readiness, customer eligibility, participants, and order lines.
4. Resolve missing information or return the request for clarification.
5. Confirm the execution products that need handoffs.
6. Move the request to ready or launch the available handoff actions.

## What Happens Next
OrdArr coordinates readiness and handoff state. Each execution product owns its accepted execution record.

## Troubleshooting
- If customer eligibility blocks triage, resolve it in CustomArr.
- If evidence is missing, use RecordArr.
