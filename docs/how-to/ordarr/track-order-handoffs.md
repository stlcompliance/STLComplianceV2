# How to track order handoffs

## Audience
Order coordinators, dispatch coordinators, customer service users, and managers

## Product
OrdArr

## Support Status
Supported by product contract/docs

## Purpose
Track whether execution products accepted, progressed, rejected, or completed OrdArr handoffs.

## Before You Start
- Order request has one or more handoffs.
- You have permission to view handoff status.

## Steps
1. Open OrdArr.
2. Open the order request.
3. Open Handoffs.
4. Review target product, handoff status, timestamps, and linked execution record.
5. Open the execution product when deeper execution detail is needed.
6. Resolve coordination, routing, or readiness exceptions in OrdArr.

## What Happens Next
OrdArr tracks order-level handoff state. Execution products remain the source of execution status, proof, exceptions, and completion.

## Troubleshooting
- If a handoff is rejected, review the rejection reason and request readiness.
- If execution status is stale, check the execution product event or read model.
