# How to create and close a CAPA

## Audience
Quality managers, quality reviewers, CAPA owners, auditors, and process owners

## Product
AssurArr

## Support Status
Supported by current UI/API

## Purpose
Create corrective and preventive action when a quality issue needs root cause analysis, assigned actions, verification, and closure.

## Before You Start
- AssurArr owns CAPA state, action plans, effectiveness verification, and closure.
- Target products own the execution work assigned to them.
- RecordArr owns retained CAPA evidence files.

## Steps
1. Open AssurArr.
2. Open CAPA.
3. Select Create CAPA.
4. Enter title, description, CAPA type, source type, severity, owner, sponsor, due date, and source references.
5. Link related nonconformances, findings, complaints, supplier issues, requirements, or records.
6. Create the CAPA.
7. Open the CAPA detail and complete root cause analysis.
8. Add action plan items and assign each action to the right person, team, or target product.
9. Track actions through assigned, in progress, completed, verified, or blocked states.
10. Add a verification plan and record effectiveness results.
11. Close the CAPA only after required actions and verification are complete.

## What Happens Next
AssurArr preserves CAPA history and publishes quality status. If verification is ineffective, revise or reopen the CAPA instead of closing it administratively.

## Troubleshooting
- If an action belongs to training, maintenance, inventory, documents, supplier follow-up, or customer follow-up, route the execution work to the owning product.
- If closure is blocked, review open actions, unresolved blockers, missing evidence, and verification state.
- If the CAPA affects a regulated requirement, use the rule/evidence owner for requirement meaning and RecordArr for retained proof.
