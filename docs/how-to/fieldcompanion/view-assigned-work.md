# How to view assigned work

## Audience
Field workers, drivers, technicians, and supervisors

## Product
Field Companion

## Support Status
Supported by current UI/API

## Purpose
Use the mobile task inbox to see product-owned work assigned to you.

## Before You Start
- Field Companion is a mobile execution surface, not the source of truth for final records.
- Assigned work must come from the owning product such as MaintainArr, LoadArr, TrainArr, RoutArr, or StaffArr.

## Steps
1. Open Field Companion.
2. Open the field inbox.
3. Review assigned task cards.
4. Use filters to narrow by status, product, due state, or task type when available.
5. Open a task to see source product, record context, instructions, and available actions.
6. Confirm you are in the correct tenant before acting.
7. Complete work in the task only when the action matches the owning product workflow.

## What Happens Next
Field Companion writes task actions back to the owning product and may use RecordArr for captured evidence.

## Troubleshooting
- If a task is missing, check the owning product assignment first.
- If a task action is unavailable, the owning product or permission gate may be blocking it.
