# How to update vendor order status

## Audience
Supply chain users and vendor portal users

## Product
SupplyArr

## Support Status
Supported by current UI/API

## Purpose
Keep the vendor order current as the vendor acknowledges, ships, backorders, or resolves exceptions.

## Before You Start
- You need access to SupplyArr Vendor orders or the configured vendor portal link.
- SupplyArr owns vendor order status; LoadArr owns receiving status after goods arrive.

## Steps
1. Open SupplyArr.
2. Open Purchasing.
3. Open Vendor orders.
4. Select the vendor order.
5. Review line items, requested dates, exceptions, and current status.
6. Use the available status, acknowledgement, update, or exception action.
7. Enter notes, dates, quantities, or exception details requested by the form.
8. Save the update.
9. Review whether the update changes receiving expectations or procurement readiness.

## What Happens Next
SupplyArr records the vendor order update and can coordinate expected receipt changes for LoadArr.

## Troubleshooting
- If the vendor used the portal, verify the portal update appears on the internal vendor order.
- If received quantities do not match, resolve the discrepancy in LoadArr receiving and then reconcile SupplyArr status.
