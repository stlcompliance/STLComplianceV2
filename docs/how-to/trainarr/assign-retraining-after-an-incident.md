# How to assign retraining after an incident

## Audience
Training admins, safety users, and managers

## Product
TrainArr

## Support Status
Intended workflow partially supported by current routes/docs

## Purpose
Turn a StaffArr incident training concern into a TrainArr remediation assignment.

## Before You Start
- StaffArr owns the incident.
- TrainArr owns retraining and remediation assignment execution.
- The current StaffArr incident create flow can flag training review; TrainArr exposes Remediation and Assignments areas.

## Steps
1. Open StaffArr and verify the incident includes the affected person and training impact.
2. Open TrainArr.
3. Open Remediation.
4. Find the incident-related remediation candidate or create the retraining assignment manually if the handoff is not present.
5. Select the person and training program.
6. Set due date, trainer, evaluator, and evidence requirements when available.
7. Create the assignment.
8. Monitor completion through TrainArr Assignments.
9. Return to StaffArr to review incident status and person readiness.

## What Happens Next
The incident remains in StaffArr. The retraining assignment, completion, and qualification outcome remain in TrainArr.

## Troubleshooting
- If no remediation candidate appears, document the StaffArr incident reference and create the assignment manually.
- If the assignment completes but readiness is unchanged, check qualification publication to StaffArr.
