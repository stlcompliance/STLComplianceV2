# How to create a certification requirement

## Audience
Training admins and compliance managers

## Product
TrainArr

## Support Status
Intended workflow partially supported by current routes/docs

## Purpose
Define what certificate, qualification, renewal, or evidence a person must maintain.

## Before You Start
- TrainArr owns certification definitions and qualification rules.
- Compliance Core may define evidence requirements, but TrainArr owns the training/certification execution.

## Steps
1. Open TrainArr.
2. Open Qualifications or Programs.
3. Create or edit the relevant qualification or requirement when the UI exposes that action.
4. Define the requirement name, who it applies to, renewal or expiration behavior, and required evidence.
5. Connect the requirement to a training program when completion should issue or renew the qualification.
6. Save the requirement.
7. Review affected people through Qualifications, Assignments, or StaffArr readiness views.

## What Happens Next
TrainArr uses the requirement to evaluate qualification status and publish readiness results to StaffArr.

## Troubleshooting
- If the UI does not expose certification creation yet, capture this as a configuration gap and use the existing qualification view for monitoring only.
- Do not create the certification as a StaffArr-only field; StaffArr may display qualification status but does not define the requirement.
