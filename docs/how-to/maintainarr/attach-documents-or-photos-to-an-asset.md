# How to attach documents or photos to an asset

## Audience
Maintenance users and records coordinators

## Product
MaintainArr

## Support Status
Intended workflow partially supported by current routes/docs

## Purpose
Connect asset evidence, photos, manuals, or inspection documents to the asset through RecordArr-backed storage.

## Before You Start
- MaintainArr owns the asset.
- RecordArr owns stored files, document metadata, retention, access history, and controlled document lifecycle.
- Final asset detail upload labels should be confirmed in the UI.

## Steps
1. Open MaintainArr.
2. Open Assets and select the asset.
3. Look for the documents, evidence, or related records area.
4. Use the available attach or upload action if present.
5. Select the document or photo and classify its purpose.
6. Confirm the file is stored or linked through RecordArr when the workflow asks for record details.
7. Review the asset detail to verify the attachment reference appears.
8. Use RecordArr for retention, access logs, packages, or controlled document review.

## What Happens Next
MaintainArr references the document in asset context. RecordArr owns the file and document controls.

## Troubleshooting
- If the asset page does not expose upload yet, create the record in RecordArr Capture and reference the MaintainArr asset as the source object.
- Do not store long-term asset files outside RecordArr when they are compliance or maintenance evidence.
