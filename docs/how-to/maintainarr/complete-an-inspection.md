# How to complete an inspection

## Audience
Technicians, inspectors, and maintenance supervisors

## Product
MaintainArr

## Support Status
Intended workflow partially supported by current routes/docs

## Purpose
Perform an asset inspection and record pass/fail or follow-up work.

## Before You Start
- MaintainArr owns maintenance inspections.
- Field Companion may be the mobile execution surface.
- The current app exposes Inspections and Templates areas; exact inspection execution controls should be confirmed.

## Steps
1. Open MaintainArr or the assigned Field Companion task.
2. Open Inspections.
3. Select the inspection to perform.
4. Review the asset, template, checklist, and due context.
5. Answer each checklist item.
6. Add notes and evidence when required.
7. Mark failed items clearly.
8. Submit or complete the inspection.
9. Create or link a defect or work order for failed items.

## What Happens Next
MaintainArr records the inspection result and any related defects or work orders. RecordArr can store evidence when documents are captured.

## Troubleshooting
- If the complete action is not available in MaintainArr desktop, use the Field Companion inspection task if assigned.
- If a failed inspection should change readiness, verify the defect or readiness update occurred.
